Pompelmi is a **TypeScript library** that adds a security layer to file upload endpoints in Node.js applications. It scans file bytes **in-process** — no HTTP call to a cloud API, no ClamAV daemon required, no file data leaving your infrastructure.

It gives you layered, composable scanning: size and MIME guards, magic-byte validation, ZIP bomb protection, structural heuristics, and optional YARA rules — all returning a consistent `clean | suspicious | malicious` verdict your application can act on immediately.

**Built for teams who cannot or choose not to send user files to a third-party scanning service** — healthcare, legal, finance, self-hosted SaaS, and privacy-first applications.

> 🧪 **Live demo:** drag a clean JPG → **CLEAN**; try the EICAR test string → **MALICIOUS**.

## Why Pompelmi

- **No cloud dependency** — scanning runs entirely in your Node.js process. Files never leave your infrastructure.
- **No daemon** — install like any npm package. No ClamAV setup, no sidecar process, no config files.
- **Defense-in-depth** — layered checks: size limits, extension allowlist, magic-byte MIME validation, structural heuristics, ZIP bomb protection, and optional YARA rules.
- **Framework-ready** — drop-in adapters for Express, Next.js, NestJS, Fastify, Koa, and Nuxt/Nitro.
- **Composable** — chain built-in scanners with your own using `composeScanners`. Control order, timeouts, and stop conditions.
- **Privacy-safe by design** — no bytes are written, forwarded, or logged unless you explicitly configure it.

## Who this is for

Pompelmi is the right choice when:

- Your app accepts file uploads from users or external systems.
- You need to block malicious uploads **without routing files through a cloud API**.
- You are in a **self-hosted, on-premises, or regulated environment** where data residency matters.
- You want a **composable, testable security layer** that integrates naturally with your existing Node.js middleware stack.

It may not be the right choice when:
- You need full antivirus coverage with daily signature updates (consider adding ClamAV via `@pompelmi/engine-clamav` alongside the heuristic layer).
- Your environment already has a trusted cloud scanning service and data residency is not a concern.

> 💡 **Testing tip:** The [EICAR test file](https://www.eicar.org/download-anti-malware-testfile/) is a safe, standard way to verify your scanner is working in development and CI. Pompelmi detects EICAR out of the box.

## Trust & maintenance

Pompelmi is actively maintained, MIT-licensed, and designed for long-term reliability. Each release includes:

- **Continuous integration** — tests run on every push across Node.js 18, 20, and 22.
- **CodeQL analysis** — static security analysis integrated into the main branch workflow.
- **SBOM** — a Software Bill of Materials is generated on every release.
- **OpenSSF Scorecard** — supply chain security posture is publicly tracked.
- **Provenance attestation** — npm releases include GitHub Actions provenance for supply chain verification.

See [SECURITY.md](https://github.com/pompelmi/pompelmi/blob/main/SECURITY.md) for the vulnerability reporting policy and response timeline.
